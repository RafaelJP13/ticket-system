# 🎫 Ticket Management System

A study project focused on learning and applying **Spec-Driven Development (SDD)** by designing and implementing a ticket management system.

## 🎯 Purpose

This repository documents my journey learning SDD by following a **specification-first** approach before writing any code.

The project covers:

- Problem definition
- Feature specifications
- Technical planning
- Task breakdown
- Implementation
- Testing

## 📁 Project Structure

```text
.
├── specs/      # Problem definition and feature specifications
├── plans/      # Technical planning
├── tasks/      # Implementation tasks
├── docs/       # Additional documentation
└── src/        # Source code
```

## 🚧 Status

This project is currently under development as part of my Spec-Driven Development learning journey.
